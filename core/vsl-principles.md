# VSL 设计原则

本 skill 的用户是视觉-空间型学习者（Visual-Spatial Learner）。Linda Silverman 的研究表明：VSL 用画面而非文字思考，整体把握而非线性推进，一旦在脑中建成正确的视觉模型，学习就是永久的。

以下原则渗透到输出的每个环节：

---

## 1. 全局优先

先看整张地图，再看某条路。每章开讲前先给出 SVG 知识地图（§1），让用户在读任何细节之前就知道有多少东西、它们之间什么关系。

## 2. 图即内容

图是理解的载体，文字是对图的批注。每个概念、流程、关系都有对应的 SVG。不要把图当作"装饰"——如果删掉图之后文字仍然完整，说明这个图画错了。

## 3. 关系优先于定义

VSL 通过"这个东西和已知的东西怎么连"来理解新概念。§3.4 的知识框架定位图、§3.3 的推导链图都在服务这一点。孤立的定义对 VSL 效率最低。

## 4. 模式 > 列表

结构化的视觉分组比平铺的文字列表更易记。对比用 HTML 表格 + 色带呈现，层级用 SVG 树形图，流程用箭头链。

## 5. 一次理解 = 永久记忆

推导不能跳步。VSL 在脑中构建画面，跳一步画面就断了，断了就丢了。宁可多写 3 段推导，也不要用一句"易得"糊弄过去。
