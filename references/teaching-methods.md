# 四种教学法的操作指南

这份文件展开 SKILL.md 里 §3 §5 §6 提到的四种教学法。每种都附了正反例。**第一次给用户讲课之前，至少把第 1 节（第一性原理）和第 3 节（苏格拉底）读完。**

---

## 1. 第一性原理（First Principles）—— Phase 3 用

### 它的本质

不是"从基础讲起"。第一性原理是**比基础还底**——退到这门学问被发明出来之前，问：当年那个人面对的是什么？他不能不假设什么？他一旦假设了某件事，下一步必然推出什么？

终点是：让最终公式 / 定理读起来像被推出来的结果，而不是空降的指令。

### 操作步骤

写 §3 时强制按这个顺序：

1. **动机**（§3.1）：优先从读者自己的体验切入（"你用 LLM 时第一个 token 为什么慢"），其次从例题倒推，最后才用学科史。
2. **最小核心**（§3.2）：把所有装饰剥掉，本章只在做**一件什么事**？一句话。
3. **逐层垒砌**（§3.3）：列出最弱假设 → 推出 → 加假设 → 推出 → 直到最终公式自然出现。**关键转折步骤**用紫色高亮 callout 标出。写完画一张 SVG 推导链图。

### 检验

写完 §3.3 后，最后一句话应该读起来像 **"……所以我们就拿到了原书 §X.Y 那个公式"**，而不是 **"作者引入了下面这个公式……"**。前者是推导出来的，后者是被宣布的。

### 正例

> **§3.1** 你用 ChatGPT 或 Claude 的时候肯定注意过：第一个 token 出来特别慢，后面就刷刷地流出来了。这不是网络延迟——这是一个叫 KV caching 的工程决策在起作用。
>
> **§3.2** 本章只在做一件事：**避免 Transformer 在生成每个新 token 时重复计算之前所有 token 的 K 和 V 矩阵**。
>
> **§3.3**
> - 事实 1：生成下一个 token 时，self-attention 需要当前 token 和**所有之前 token** 的 K、V。
> - 事实 2：之前 token 的 K、V 在上一步已经算过了，它们不会因为新 token 的加入而改变。
> - **关键一步**：既然旧的 K、V 不变，那就没必要重算——存起来，下一步直接读。
> - 这就是 KV cache。所以我们就拿到了原书那个"prefill + decode"的两阶段流程。

### 反例（不要这样写）

> **§3** KV cache 是 Transformer 推理中的一种优化技术，通过缓存之前时间步的 Key 和 Value 来避免重复计算。

这是教科书摘要——**宣布**结论而不是**推出**结论。

### 当原书顺序和第一性原理冲突时

少数情况下，原书先讲"什么是 X"，再讲"为什么需要 X"。在 §3 按第一性原理重排，**§4 回到原书顺序逐字讲**。两个阶段目的不同。

## 2. 逆向学习法 —— Phase 2 用

### 操作步骤

1. 读完整章后，找到所有**例题、习题、PPT 练习、明示目标**。
2. 翻译成能力陈述：**"给定 [输入]，能 [动作]，并且 [质量约束]"**。
3. 列 3–6 条，必须对应到原书具体出处。

### 正例

```
1. 给定一个二元一次方程组（系数任意），能在 5 步内用代入消元法求出唯一解。（对应例题 3.2）
2. 当方程组有 3 个以上未知数时，能识别"应改用矩阵方法"。
```

### 反例（禁止使用的动词）

"理解、掌握、了解、熟悉"——这些动词不可证伪。用"能算出"、"能在 N 分钟内"、"能给出反例"、"能识别哪些情形不适用"这种**有动作、有约束**的句式。

## 3. 苏格拉底诘问 —— Phase 6 用

### 出题原则（必须遵守）

1. **不出"X 的定义是什么"**——背得出不代表懂。
2. **每题都要求用户使用概念，而不是描述概念。**
3. **至少有一道'故意挖坑'题。** 看似合理但违反某个假设。
4. **每题答案预测错答模式。** "如果你答错了，最可能是因为你把 A 和 B 混了，回去看 §Y.Z"。
5. **难度递进**：Q1 戳 §3 核心 → Q2 戳 §4 细节 → Q3+ 边界 / 反例。

### 题量

4–8 题。少于 4 覆盖不够，多于 8 用户会跳过。

### HTML 实现

用 `<details><summary>` 折叠答案（见 `html-design.md` §5）。答案区域绿底，错答模式红色虚线分隔。

### 正例

> **Q1**：我们推出 KV cache 时，核心假设是"旧 token 的 K、V 不变"。如果模型架构换成双向注意力（像 BERT），这个假设还成立吗？KV cache 还能用吗？
>
> **答**：不成立。双向注意力里每个 token 的表示依赖所有其他 token，包括后面的。加一个新 token 会改变之前所有 token 的 K、V。所以 KV cache 对 BERT 不适用——它只在自回归（单向）解码时有效。如果你答"还能用"，你把单向和双向注意力混了。

## 4. 费曼讲法 —— Phase 5 用

### 操作要点

1. **彻底放下书的术语。** "哈密顿量" → "系统的总账本"。
2. **用日常物件、生活场景、对话体。**
3. **故意夸张到不严谨。** 严谨是 §4 的事。
4. **结尾必须自我打假。** 列出"大白话和原书的差别"——哪些地方为了好懂牺牲了精确？这些差别在做题时会怎么咬人？

### 反例

> 哈密顿量是描述系统总能量的算符，它作用在波函数上给出系统的演化。

这是术语翻译，离费曼讲法还差一层。如果你写不出像样的类比，回 §3 重写——大概率你那里就没真搞懂。

## 5. 四种方法之间的协作

这四种方法一起构成一条认知链：

```
逆向（§2）：知道往哪去
   ↓
第一性原理（§3）：知道每一步为什么是这一步
   ↓
逐字精读（§4）：把每一步落到原书具体文字上
   ↓
费曼（§5）：用大白话压力测试自己的理解
   ↓
苏格拉底（§6）：把测试交给用户做
   ↓
闭环（§7）：用清单核验整条链上有没有断点
```

任何一环弱化或省略，整条链就漏。这就是为什么 SKILL.md 里"不许跳"——每一步都有它在认知链上的位置，跳了就断。
