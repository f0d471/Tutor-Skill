# 认知科学依据

本文档记录 Tutor-Skill 设计的认知科学依据。每条规则都可以追问一句"这个规则保护的是什么认知原则？"——答案在这里。

---

## 1. Mayer 多媒体学习认知理论（CTML）

15 条原则，基于 200+ 实验得出。按三种认知加工类型分组，标注每条在本 Skill 中的落地位置。

### 1.1 减少无关加工 → Forbidden 列表的来源

| 原则 | 含义 | Tutor-Skill 落地 |
|------|------|------------------|
| **连贯性（Coherence）** | 去掉无关文字、图片、声音 | Forbidden 列表：禁止装饰性 emoji、动画光晕 |
| **信号（Signaling）** | 用编号、高亮、箭头突出关键结构 | **新增**：推导链步骤编号（Step 1/n），关键转折紫色高亮 |
| **冗余（Redundancy）** | 动画+解说+字幕 三者叠加不如只留两个 | Forbidden 列表：不大段复述原书；同一件事不换说法重复 |
| **空间邻近（Spatial Contiguity）** | 图紧挨在引出它的文字旁边 | 规则 5：图紧跟引出句，不堆多张图 |
| **时间邻近（Temporal Contiguity）** | 旁白和画面对应同步 | 不适用（静态课件） |

### 1.2 管理必要加工 → 七阶段设计的来源

| 原则 | 含义 | Tutor-Skill 落地 |
|------|------|------------------|
| **分段（Segmenting）** | 知识按学习者节奏分批呈现 | 规则：单次输出不超 3 个 Phase，写完暂停等用户确认 |
| **预训练（Pre-training）** | 先命名关键概念，再深入讲解 | §1 知识清单 + 每个概念附一句话定义 |
| **模态（Modality）** | 用双通道（视+听）比单通道好 | §4 图文交织——图不是装饰，是理解的载体 |

### 1.3 促进生成加工 → 教学交互的来源

| 原则 | 含义 | Tutor-Skill 落地 |
|------|------|------------------|
| **个性化（Personalization）** | 对话式风格比正式风格学习效果更好 | §5 费曼讲法可选用对话体（"你"、"咱们"），但不强制 |
| **生成活动（Generative Activity）** | 主动输出优于被动接受 | §6 苏格拉底诘问 + §7 闭环验真——必须做题、必须合书复述 |
| **引导发现（Guided Discovery）** | 引导学习者自己发现规律，不直接告知 | §3 第一性原理推导——引向结论但不宣布结论 |
| **具体化（Concreteness）** | 先给具体实例再给抽象表述 | §3.1 从读者体验切入（"你用 LLM 时第一个 token 为什么慢"） |

## 2. PEBBLE 教学评估框架

NeurIPS 2025 基准，五个维度评估 AI 辅导的教学质量。**所有 LLM 在"诊断"和"误解修复"维度上都有系统性缺陷——这是 Tutor-Skill 的护城河方向。**

| 维度 | 含义 | Tutor-Skill 当前 | 本轮改进 |
|------|------|------------------|----------|
| **脚手架（Scaffolding）** | 逐步引导，不直接给答案 | ✅ §3 推导链引向结论但不宣布 | — |
| **诊断性提问（Diagnostic Q）** | 探测学生哪里没懂、为什么没懂 | ❌ 缺失 | §6 课前误解诊断：出题前先列 3-5 个本章最常见误解 |
| **误解修复（Misconception Repair）** | 指出典型错误并对比正确理解 | 🟡 §4 有正例反例，但不针对常见误解 | 每题答案强化错答模式分析："如果你选了 B，你可能混淆了 X 和 Y，回去看 §Z" |
| **元认知支持（Metacognition）** | 计划、监控、反思提示 | 🟡 §2 逆向目标 + §7 闭环，中间阶段缺失 | §3→§4、§4→§5 过渡处加"暂停思考"框 |
| **情感支持（Affect）** | 鼓励、归属感、动机维持 | 🟡 风格偏严谨 | 个性化原则覆盖（可选对话式风格） |

## 3. AI 辅助自我调节学习（SRL）—— 元分析证据

32 项研究、3029 名被试的综合元分析（2025）：

| 发现 | 效应量 | 对 Tutor-Skill 的启示 |
|------|--------|----------------------|
| AI 对 SRL 总体效果 | **g = 1.613**（大效应） | 方向正确——AI 辅助调节学习有显著收益 |
| AI 对"前思阶段"效果最强（目标设定、计划） | — | §2 逆向目标（"学完能干什么"）策略正确 |
| AI 对"执行阶段"效果不稳定，需要上下文感知 | — | 静态课件缺少实时交互，短期不追求自适应 |
| 对话式 AI 系统始终优于自适应/分析式系统 | — | Tutor-Skill 是静态课件，不追求实时对话，但 §6 的问答设计是折中 |
| **"过早给答案"会显著削弱学习效果** | — | §3 推导链设计正确：引向结论但不宣布结论 |
| 干预时间越长，效果越强 | — | 多章节使用积累效果，跨章导航和进度统计有价值 |

## 4. 教学等价假设

Frontiers in Psychology (2025)，比较不同程度的视觉丰富度对学习结果的影响：

**核心发现**：内容和教学法足够好时，多媒体设计的丰富度对学习结果**无显著影响**（F = 0.52, p = 0.60）。但丰富设计显著提升了学习者的主观体验和参与动机。

### 对 Tutor-Skill 的启示排序

```
最重要的： 内容准确性 + 教学法（Forbidden 列表 + 推导链 + 苏格拉底 + 闭环）
次要的：   Forbidden 列表的视觉约束（保持整洁即可——禁止渐变、禁止光晕）
锦上添花： CSS 动画、渐变、花哨的视觉效果
```

**这不是说视觉不重要，而是说不要在 CSS 上花的精力大于内容。** 当前 Forbidden 列表的"禁止渐变文字、禁止动画光晕"已经足够——它们保护的是"不让视觉干扰认知"，而不是"不让视觉好看"。
