# 硬性规则

agent 填充课件时必须遵守以下规则。任何一条违反都是失格。

---

## 内容规则

1. **节奏贴着书。** 书外补充用 callout 标出，出现频率应低于原书内容。禁止把书外内容混进原文引用。
2. **原书有图就嵌进来。** 章里有图，课件里就要有图。路径从原书 md 的相对路径改写过来。
3. **每次都先列知识点清单。** §1 是稽核线，先列再讲。
4. **该长就长，但长度 = 深度 × 覆盖面。** 不是同一件事换说法重复。单章 HTML 课件 8000–15000 字是合理范围。
5. **段落要短。** 技术讲解部分 1–3 句话换段。图紧跟在引出它的句子后面。
6. **第一性原理优先于书的表面顺序。** §3 用第一性原理排，§4 回到书的顺序。
7. **不伪造引用、页码、章节号。**
8. **没有跨会话记忆。** 磁盘上 `output/` 和 `notes/` 是唯一持久状态。
9. **不省略 §6（苏格拉底）和 §7（闭环）。**
10. **不在深度上偷懒。** 判断标准：删掉某段后读者理解是否断点。是→不能删。否→一开始就不该有。
11. **每个概念都画出它在知识地图上的位置。** SVG 树形图 / 对照表，标出上游、下游、邻居。

---

## 执行规则

12. **单次输出不超 3 个 Phase。** 一章 7 个阶段分批生成，每次最多写 3 个 Phase 的内容，写完暂停等用户确认再继续。避免上下文爆满导致质量塌方或 agent 卡死。
    - 推荐分批：§1–§3 → §4 → §5–§7
    - 如果单章内容量大（知识点 > 20），进一步缩小批次：§1–§2 → §3–§4 → §5–§7
13. **每次暂停时告诉用户下一步做什么。** 不要让用户猜——明确说"§1–§3 已完成，确认后我继续写 §4"。

---

## Forbidden 列表（强制禁止）

### 文风层

| 禁止 | 替代 |
|---|---|
| "不是 X 而是 Y"对偶句式 | 直接说 X 是什么、Y 是什么 |
| "稳稳/牢牢/妥妥"等副词 | 删掉，不需要修饰 |
| "让 __ 显得不可避免"拟人修辞 | 说清楚为什么必须这么做 |
| "让我们一起"煽动开场 | 直接开始讲 |
| 段落里装饰性 emoji | emoji 只用在清单标记（🎯）和 callout 标题 |
| 一段超过 2 个破折号 | 拆成短句 |
| "显然/易得/不难看出" | 老老实实写推导步骤 |

### 视觉层

| 禁止 | 替代 |
|---|---|
| 渐变文字（`background-clip: text`） | 纯色文字 |
| emoji 做标题前缀 | 用 callout-title 里的图标即可 |
| 动画光晕 / 发光效果 | 用 `box-shadow` 做静态深度 |
| 所有模板用同一个色板 | 按内容类型选模板（暖色/冷色/对比色） |

### 内容层

| 禁止 | 替代 |
|---|---|
| 只给定义不给例子 | 每个定义至少配一个具体例子 |
| 只给例子不给反例 | 重要概念同时给正例和反例 |
| 跳过前置知识直接讲结论 | 先交代需要什么前置，再推结论 |
| 空洞类比（"像一个盒子"、"就像管道"） | 类比必须能推出现象，否则不用 |
| 用术语解释术语（哈密顿量是"系统的能量算符"） | 费曼讲法（§5）必须用大白话 |

**允许的例外**：§5 结尾的"大白话 vs 原书差别表"是唯一允许的对比表格，因为它承载了元认知校验功能，不可用段落替代。

## Slop Test（AI 味检测）

写完课件后自问：

- 一个学完这章的学生，能不能用课件里的内容**做对原书的习题**？不能 = 质量不达标。
- 如果把课件里的所有 callout 删掉，剩下的原书内容还能不能独立成章？不能 = 对原书依赖过重。
- 一个同行看到这个课件，会不会觉得"这是 AI 批量生产的"？会 = 设计偷懒了。
