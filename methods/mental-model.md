# 心智模型构建（Phase 3 用）

## 本质

§3 不是讲学科史，不是讲公理化动机，不是列概念清单。§3 的任务是：**在学生脑子里种一棵可以挂东西的树。**

学生读完 §3 时，应该能用自己的话说出：
1. 这章**最核心的一个概念**是什么
2. 这个概念的**操作直觉**是什么（一句话可执行）
3. 本章其他关键概念**怎么从这个核心长出来**

如果学生说不出这三点，§3 失败了。

## 三步结构

### §3.1 核心概念锚定（1-2 句话）

**不做**：学科史叙事（"19 世纪数学家发现……"）、公理化动机（"直觉可能骗人……"）

**做**：直接告诉学生这章最核心的一个概念是什么。用世界知识解释，不用原书语言。

**标准**：
- 一句话，不超过 25 个字
- 不包含多于 1 个术语
- 学生读完能回答"这章在讲什么"

**示例**：

| 章节 | 核心锚定 |
|------|----------|
| 概率论·基本概率定律 | 概率 = 给"不确定性"分配一个 0 到 1 的数 |
| 概率论·条件概率与贝叶斯 | 知道了新信息后，概率要更新 |
| 概率论·容斥原理 | 加了重复的，就减回来 |
| CPU 设计·第一节课 | CPU 的工作 = 取指→译码→执行→写回 |
| 线性代数·矩阵乘法 | 矩阵乘法 = 对空间做变换 |

### §3.2 操作直觉（一句话可执行）

**不做**：给定义（"容斥原理是……"）、给公式（Pr(A∪B) = ……）

**做**：给一个**操作层面的直觉**，学生看到这个概念就能用、就能算、就能判断。

**标准**：
- 一句话，动词开头
- 读完能直接操作，不需要额外知识
- 对应到具体场景

**示例**：

| 概念 | 操作直觉 |
|------|----------|
| 容斥原理 | 算两个集合并的大小：先加，再减掉重叠的部分 |
| 独立性 | 一个发生了，不影响另一个的概率——直接乘 |
| 贝叶斯 | 知道结果了，反推原因：用全概率归一化 |
| 条件概率 | 缩小范围：只看 B 发生的那些情况里 A 发生的比例 |
| 全概率公式（对立事件） | 正面算不出来，反面算完用 1 减 |
| KV cache | 旧的 K、V 不变，存起来直接读，别重算 |
| 握手机制 | 上游发 valid，下游发 ready，同时为 1 才传数据 |

### §3.3 心智模型生长（从核心展开到全章）

**不做**：按学科发展时间线排列（"罗素悖论 → 集合论 → 公理化"）、按原书小节顺序复述

**做**：从 §3.1 的核心概念出发，**用第一性原理推导**展开到本章所有关键概念。每一步：
1. 回到核心概念建立连接（"刚才说概率是量化不确定性，那怎么量化？"）
2. **用第一性原理推导**：不宣布下一个概念，而是问"已有的东西够用吗？不够用的话，必须发明什么？"——让每个新概念像被逼出来的，不是被引入的
3. 给操作直觉（§3.2 已给的直接引用，新概念当场给）
4. 用世界知识解释（不是用原书语言）

**第一性原理推导的要点**（详见 `methods/first-principles.md`）：
- 每一步问：**现有工具解决不了什么问题？** → 这个问题逼出了什么新概念？
- 关键转折：当现有工具真的不够用时，标注"🔑 关键转折"——这是新概念诞生的时刻
- 推导链的终点应该是"所以这个概念不可避免地出现了"，而不是"作者引入了下面这个概念"
- 不展开公式用法和做题——那是 §4 的事。§3.3 止于"概念诞生的那一刻"

**结构**：用推导链（flow-container 或 SVG）展示概念生长路径。每个节点：
- 节点名 = 概念名
- 节点描述 = 一句话操作直觉
- 箭头 = **第一性原理推导**（"现有工具不够用，必须发明 B"）

**示例（概率论·基本概率定律）**：

```
核心：概率 = 量化不确定性（0 到 1 的数）
  ↓ 怎么算？等可能时直接数，但不等可能呢？现有工具不够用
公理化：必须给概率一个严格的定义——三条公理兜底（非负、规范、可加）
  🔑 关键转折：没有公理，"概率"只是一个直觉，无法处理无穷和连续情况
  ↓ 有了公理，两个事件怎么算？直接加会重复
容斥：加了重复的减回来 + 文氏图一眼看出
  ↓ 知道一个事件，另一个怎么算？不需要新工具
互补：正面算不出，反面算完用 1 减（公理的直接推论）
  ↓ 知道新信息后概率要更新，但现有工具处理不了"缩小范围"
条件概率：缩小范围再算——必须引入"已知 B"这个新概念
  🔑 关键转折：没有条件概率，就无法处理"部分信息"的情况
  ↓ 知道结果反推原因，方向反过来
贝叶斯：果→因的反推——条件概率 + 全概率的组合
```

**示例（CPU 设计·第一节课）**：

```
核心：CPU = 取指→译码→执行→写回
  ↓ 指令从哪来？必须有个地方存
内存：指令和数据都在内存里，按地址取
  ↓ 取到了怎么确认传完了？不能假设对方收到了
握手：上游 valid，下游 ready，同时为 1 才传数据
  🔑 关键转折：没有握手，就无法保证数据传输的可靠性
  ↓ 跳转怎么办？顺序执行不够用了
PC：程序计数器指向当前指令地址，跳转就是改 PC
```
核心：概率 = 量化不确定性（0 到 1 的数）
  ↓ 怎么算？
操作：数有利情况 ÷ 总情况（等可能时）
  ↓ 不等可能怎么办？
公理化：三条公理兜底（非负、规范、可加）
  ↓ 两个事件怎么算？
容斥：加了重复的减回来 + 文氏图一眼看出
  ↓ 知道一个事件，另一个怎么算？
互补：正面算不出，反面算完用 1 减
```

  ↓ 指令从哪来？

**要求**：
- 每个节点之间有明确的认知逻辑连接词（"怎么算"、"怎么办"、"然后呢"）
- 节点数 ≤ 8（太多记不住）
- 最后一个节点应该覆盖本章的核心内容（不一定是全部，但至少是 §4 精读的主要部分）

### §3.4 在知识框架中的位置（保留）

SVG 树形图或对照表，展示本章在学科中的位置。上游（依赖什么）、下游（打开什么）、邻居（并列什么）。

### §3.5 暂停思考（改造）

问题改为针对心智模型，不是针对推导链：

```
> 🧠 **暂停思考**
>
> 现在合上书（或者滚到屏幕外面去），问自己：
> - 你能用自己的话说"这章最核心的概念是什么"吗？
> - [本章核心概念]的操作是什么？（一句话）
> - §3.3 的生长链里，哪个节点你觉得"为什么要这样"？
>
> 如果说不出来，把对应节点重新读一遍再继续。
> —— §4 会用原书的细节来填充这个框架。但先确保骨架是稳的。
```

## 检验

- §3.1 读完后，学生能用一句话回答"这章讲什么"
- §3.2 读完后，学生看到概念能直接操作（不需要回去翻定义）
- §3.3 读完后，学生脑子里有一棵完整的"树"——知道本章有哪些关键概念、它们之间什么关系
- §3.3 的生长链，每个箭头都是第一性原理推导（"现有工具不够用，必须发明 B"），不是简单连接
- §3.3 最后一个节点应该读起来像"所以这个概念不可避免地出现了"，而不是"作者引入了这个概念"
- 学生读完 §3 的感受是"我知道这章要干什么了"，而不是"我知道这章有什么了"（那是 §1）或"我知道为什么需要这个了"（那是旧 §3）

## 正例

> **§3.1** 概率的核心：给"不确定性有多大"分配一个 0 到 1 的数。0 = 不可能，1 = 必然。
>
> **§3.2** 操作直觉：等可能的时候，数一数有利情况，除以总情况，就是概率。不等可能？三条公理兜底。
>
> **§3.3**
> - 核心：概率 = 0 到 1 的数
> - 怎么算？等可能时：数 ÷ 总数。不等可能时：公理化定义
> - 两个事件？容斥原理——加了重复的减回来，文氏图一眼看出
> - 知道一个事件，另一个？互补——1 减去对立面
> - 知道新信息要更新？条件概率——缩小范围再算
> - 知道结果反推原因？贝叶斯——果→因

## 反例（不要这样写）

> **§3** 概率论的发展经历了从古典概率到公理化概率的历程。19 世纪，柯尔莫戈洛夫提出了三条公理，奠定了现代概率论的基础……

这是学科史叙事——学生读完不知道"这章要干什么"，只知道"这章的历史背景"。

> **§3** 本章包含以下概念：结果空间、概率公理、全概率公式、容斥原理、文氏图……

这是概念清单——那是 §1 的事，不是 §3。

## 与旧版"第一性原理"的关系

旧版 §3 讲"为什么需要这个概念"（学科发展逻辑）。新版 §3 讲"这个概念的核心是什么，怎么从核心长出全章"（认知逻辑）。

两者不矛盾：好的心智模型自然包含了"为什么需要"——当你问"不等可能怎么办？"时，公理化的必要性就自然出来了。但出发点不同：旧版从学科出发，新版从学生出发。
