# `/tutor verify` — 校验模式

## 参数

- `$@` — 可选：指定课件文件名，不指定则取 output/ 最新的

## 流程

### 1. 定位课件和原文

```
1. ls output/
2. 如果 $@ 指定了文件 → 找到对应 .html
3. 如果没有参数 → 取 output/ 中最新的 .html 文件
4. ls raw/ → 找到对应的源文件（.md 或 .pdf）
5. 如果找不到对应原文 → 提示用户确认
```

### 2. 读规则

Read `core/content-integrity.md`（需要知道什么是"准确"）和 `core/quality-standards.md`（需要知道 Forbidden 列表）。

### 3. 逐条校验

从课件中提取以下类型的内容，逐条对比 raw/ 原文：

| 校验项 | 怎么查 | 不通过怎么处理 |
|---|---|---|
| **原文引用**（`<blockquote class="book-quote">`） | 在 raw/ 中搜索对应文字 | 标记差异，修正引用 |
| **章节号 / 公式编号** | 对照 raw/ 目录结构 | 修正编号 |
| **图片路径** | 检查 `<img src>` 是否对应 raw/ 中的文件 | 修正路径 |
| **书外补充**（`callout-supplement`） | 确认标了"书外补充" | 未标的补上 callout |
| **PPT 考点**（`callout-ppt`） | 确认标了"PPT 补充" | 未标的补上 callout |
| **公式内容** | 用 KaTeX 渲染后对比原文公式 | 修正 LaTeX |
| **知识点遗漏** | 对照 raw/ 目录，看课件 §1 清单是否完整 | 标记遗漏项 |

### 4. 修正

- **就地修正**：直接修改 output/ 中的 HTML 文件
- **不删除内容**：只修正错误，不因为"多余"而删段落
- **不添加内容**：只校验已有内容的准确性，不补新知识点

### 5. 输出校验报告

```
## 校验报告

### 通过（N 条）
- ✅ 原文引用 §4.2 准确
- ✅ 公式 (3.1) 准确
...

### 修正（M 条）
- ❌→✅ 原文引用 §4.3："原文A"→"原文B"（原文实际是...）
- ❌→✅ 图片路径：`../fig/3.1.png` → `raw/fig/3.1.png`
...

### 遗漏（K 条）
- ⚠️ raw/ §4.5 有一段未在课件中出现
...
```

- 提示用户打开修正后的 HTML 确认
- 如果修正项多，建议重新讲课
