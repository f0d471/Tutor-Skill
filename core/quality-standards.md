# 质量标准

课件的质量底线——什么算不合格、怎么自查。三层递进结构。

---

## 第一层：Forbidden 列表（17 条强制禁止）

### 文风层（7 条）

> 为什么：AI 味是课件可信度的最大杀手。用户读到"让我们一起探索……"就知道是 AI 写的，立刻关掉。文风层禁止的就是 AI 生成文本的 7 个最典型指纹。

| # | 禁止 | 替代 |
|---|------|------|
| 1 | "不是 X 而是 Y"对偶句式 | 直接说 X 是什么、Y 是什么 |
| 2 | "稳稳/牢牢/妥妥"等副词 | 删掉，不需要修饰 |
| 3 | "让 __ 显得不可避免"拟人修辞 | 说清楚为什么必须这么做 |
| 4 | "让我们一起"煽动开场 | 直接开始讲 |
| 5 | 段落里装饰性 emoji | emoji 只用于清单标记（🎯）和 callout 标题 |
| 6 | 一段超过 2 个破折号 | 拆成短句 |
| 7 | "显然/易得/不难看出" | 老老实实写推导步骤 |

### 视觉层（4 条）

> 为什么：教学等价假设表明——内容和教学法足够好时，多媒体丰富度对学习结果无显著影响（F=0.52, p=0.60）。花哨的视觉分散注意力。保持干净即可。

| # | 禁止 | 替代 |
|---|------|------|
| 8 | 渐变文字（`background-clip: text`） | 纯色文字 |
| 9 | emoji 做标题前缀 | 用 callout-title 里的 CSS 图标即可 |
| 10 | 动画光晕 / 发光效果 | 用 `box-shadow` 做静态深度 |
| 11 | 所有模板用同一个色板 | 按内容类型选模板（暖色/冷色/对比色） |

### 内容层（6 条）

> 为什么：CTML 连贯性原则（去掉无关内容）+ 脚手架原则（不给答案给梯子）+ 具体化原则（类比必须能推出现象）。这是最核心的一组——违反任何一条直接伤及学习效果。

| # | 禁止 | 替代 |
|---|------|------|
| 12 | 只给定义不给例子 | 每个定义至少配一个具体例子 |
| 13 | 只给例子不给反例 | 重要概念同时给正例和反例 |
| 14 | 跳过前置知识直接讲结论 | 先交代需要什么前置，再推结论 |
| 15 | 空洞类比（"像一个盒子"、"就像管道"） | 类比必须能推出现象，否则不用 |
| 16 | 用术语解释术语 | 费曼讲法（§5）必须用大白话 |
| 17 | 书外内容混入原文引用 | 书外补充用蓝色 callout，PPT 补充用橙色 callout |

### 例外条款

§5 结尾的"大白话 vs 原书差别表"是唯一允许的对比表格。理由：它承载了元认知校验功能——让读者意识到哪里为了好懂牺牲了精确。不可用段落替代。

## 第二层：Slop Test（3 条 AI 味检测）

### S1：学生能用课件做对原书习题？

**依据**：CTML 生成活动原则——学习的证据是能"用"，不是能"复述"。PEBBLE 脚手架维度——课件必须支撑到解题能力。

**判定方法**：
1. 随机抽本章一道原书例题
2. 合上原书，只用课件
3. 能独立做出来 → ✅
4. 需要翻原书查公式/查步骤 → ❌ 说明课件缺了关键推导步骤

**修正方向**：缺什么补什么。不要加"总结"，加推导步骤。

### S2：删掉所有 callout 后，原书内容能独立成章？

**依据**：callout 是补充，不是拐杖。如果删掉 callout 后内容断裂，说明 Agent 在 callout 里藏了本该在正文里的信息。

**判定方法**：
1. 用文本编辑器删除所有 `callout` 类元素
2. 从 §1 读到 §7
3. 有没有地方概念突然出现却没被定义？→ ❌ 那个 callout 里藏了定义
4. 有没有推导断在半路？→ ❌ 那个 callout 里藏了推导步
5. 读起来流畅完整 → ✅

**修正方向**：把藏在 callout 里的关键信息移到正文，callout 只保留"额外补充"。

### S3：同行会不会觉得"这是 AI 批量生产的"？

**依据**：AI 味 = 失去信任。教学不是信息传递，是建立信任关系。同行能一眼认出 AI 味道的东西，学生早晚也能。

**判定方法（同行视角，逐条检查）**：

| 检查项 | 得分规则 |
|--------|---------|
| 有对偶句式（"不是 X 而是 Y"） | AI 味 +1 |
| 段落长短均匀得像排版 | AI 味 +1 |
| 每段以相似句式开头（"值得注意的是"、"需要强调的是"……） | AI 味 +3 |
| 没有具体例子，全是抽象描述 | AI 味 +2 |
| 推导链每步都有清晰动机和过渡 | AI 味 -1 |
| 例题选了有代表性的具体数据而不是泛泛而谈 | AI 味 -1 |

**阈值**：
- 得分 ≥ 3 → ❌ 需要大修
- 得分 1–2 → 🟡 有风险，建议针对性修改
- 得分 ≤ 0 → ✅ 通过

## 第三层：结构自检（三层递进）

顺序不可颠倒：局部 → 上下文 → 全局。

### [局部自检] 逐 section 检查

每个 section 独立检查：
- **内容完整性**：该 section 声称讲的知识点都讲完了？
- **引用准确**：原文引用是否可追溯到材料？
- **格式合规**：callout 类型正确？图紧接引出句？
- **正反例匹配**：定义 → 正例 → 反例 三步齐全？
- **步骤编号**：推导/例题每个步骤有序号（Step 1/n）？

### [上下文自检] 检查 section 间过渡

Section 之间的连接检查：
- **连贯性**：section A 结尾和 section B 开头是否有逻辑断裂？
- **一致性**：同一个符号/术语在不同 section 是否含义一致？
- **覆盖**：§1 清单里的知识点在 §3–§5 中全部出现？
- **元认知提示**：§3→§4、§4→§5 过渡处有"暂停思考"框？
- **苏格拉底覆盖**：§6 的题目是否覆盖 §1 的知识点？

### [全局自检] 检查整体质量

整份课件作为整体检查：
- **风格统一**：全部 7 个 Phase 视觉一致，不因分 3 批生成而风格割裂？
- **布局平衡**：§1–§7 篇幅比例合理？§4 应该最长（通常占 40–50%）
- **导航可用**：跨章导航链接可正常工作？
- **进度统计**：进度数字准确？（与实际 `output/` 文件数一致）
- **HTML 有效性**：`scripts/validate-html.mjs` 全部通过？
- **Slop Test**：全部三问 ✅？

写完课件后按此格式输出自查结果。必须逐条，不能跳过。

...

### 第二层：Slop Test

3. 同行不觉得是 AI 批量生产？ ✅ — 得分 -2

### 第三层：结构自检
[局部] §1 ✅ §2 ✅ §3 ✅ §4：❌ ReLU 段缺反例
[上下文] §3→§4 过渡 ✅ §4→§5 缺元认知提示 ❌
[全局] 风格统一 ✅ 布局比例 ✅ 导航 ✅ validate-html ✅
