# 设计哲学

## 根本张力

Skill 设计存在一个根本张力：规定多少、自由多少。

| 极端 A | 极端 B |
|--------|--------|
| 把 Agent 当脚本执行器：每一步怎么写、用什么词、放什么图都规定死 | 只给一个模糊目标："你做一份好课件"，Agent 完全自主 |
| 优点：输出一致性高 | 优点：适应性强，不同学科不同教法 |
| 缺点：僵化，好的数学讲法和好的历史讲法完全不同 | 缺点：输出质量不稳定，AI 味重 |

## Tutor-Skill 的选择

不走两个极端。**把约束放在"不能做什么"上，把自由放在"怎么做"上。**

```
┌─────────────────────────────────────┐
│         自由空间（最大）              │
│  Agent 决定：用什么比喻、怎么组织段落、 │
│  画什么图、例题选哪个、推导链怎么拆    │
├─────────────────────────────────────┤
│         质量门槛（中间层）             │
│  每个 Phase 的质量标准："读完后感        │
│  觉像推导出来的，不是被宣布的"          │
│  通过 Slop Test 才是合格的课件         │
├─────────────────────────────────────┤
│         硬性约束（最小）              │
│  Forbidden 列表：17 条禁止项           │
│  七阶段顺序不可调换                    │
│  节奏贴着书走、不伪造引用              │
└─────────────────────────────────────┘
```

## 五条核心原则

### 1. 教无定法，但有定界

物理课的"第一性原理推导"和历史课的"因果链叙事"本质不同，Agent 必须根据学科特点自由组织。但无论什么学科，禁止 AI 味（对偶句式、空洞类比、宣布结论不推导）是共通的。

### 2. 自由越大，门槛越硬

自由空间越大，质量验证就必须越严格。Slop Test 的三问就是自由之后的保险丝：如果自由导致学生做不对题、导致原书内容被稀释、导致同行看出 AI 味——那就是自由过头了。

### 3. 骨骼不许动，血肉可以换

七阶段顺序是认知科学的结论（先有全局视野 → 再建立动机 → 再推导 → 再精读 → 再费曼 → 再自测 → 再闭环），不许调换。但每个阶段怎么展开——用什么案例、怎么画图、怎么过渡——Agent 自己判断。

### 4. 禁止比规定更稳

告诉 Agent"禁止做 X"比"你必须做 Y"更稳定，因为 Y 在不同场景下可能不合适，但 X 几乎总是错的。"禁止空洞类比"比"必须用生活类比"更稳——有些概念不需要类比，强加反而误导。

### 5. Skill 是护栏，不是铁轨

护栏告诉你哪里会掉下去（Forbidden 列表、质量门槛），铁轨替你做选择。Tutor-Skill 选护栏模式，因为教学这件事，好的选择太多了，替 Agent 规定唯一路径是自断手脚。

### 6. 课件是原书的完整替代品，不是摘要

§4 的目标不是"总结原书的要点"，而是"学生读完 §4 就不需要再打开原书"。这意味着：
- 原文引用要足够多，组合起来覆盖原书全部内容
- 解说要足够详细，不能吝啬——宁可多讲几句，不要让学生觉得"还是得去看原书"
- 图景解说不是 1-2 句话的概括，而是像站在学生旁边详细展开

## 与其他模块的关系

- `learning-science.md`：护栏的认知科学依据——每条规则保护了什么认知原则
- `content-integrity.md`：硬性约束层——12 条内容规则（能做什么、不能做什么）
- `execution-protocol.md`：中间层——怎么做（分批、暂停、验证、断点、有界生成）
- `quality-standards.md`：质量门槛——Forbidden 列表 + Slop Test + 三层自检
- `phases.md`：自由空间的结构骨架——七阶段定义 + 质量标准（不做具体规定）
