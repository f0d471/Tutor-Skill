# 苏格拉底诘问（Phase 6 用）

## 出题原则（必须遵守）

1. **不出"X 的定义是什么"**——背得出不代表懂。
2. **每题都要求用户使用概念，而不是描述概念。**
3. **至少有一道'故意挖坑'题。** 看似合理但违反某个假设。
4. **每题答案预测错答模式。** "如果你答错了，最可能是因为你把 A 和 B 混了，回去看 §Y.Z"。
5. **难度递进**：Q1 戳 §3 核心 → Q2 戳 §4 细节 → Q3+ 边界 / 反例。

## 题量

4–8 题。少于 4 覆盖不够，多于 8 用户会跳过。

## HTML 实现

用 `<details><summary>` 折叠答案（见 `renderers/html-shell.md`）。答案区域绿底，错答模式红色虚线分隔。

## 正例

> **Q1**：我们推出 KV cache 时，核心假设是"旧 token 的 K、V 不变"。如果模型架构换成双向注意力（像 BERT），这个假设还成立吗？KV cache 还能用吗？
>
> **答**：不成立。双向注意力里每个 token 的表示依赖所有其他 token，包括后面的。加一个新 token 会改变之前所有 token 的 K、V。所以 KV cache 对 BERT 不适用——它只在自回归（单向）解码时有效。如果你答"还能用"，你把单向和双向注意力混了。
