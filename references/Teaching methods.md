# 四种教学法的操作指南

这份文件展开 SKILL.md 里 §3 §5 §6 提到的四种教学法。每种都附了正反例。**第一次给用户讲课之前，至少把第 1 节（第一性原理）和第 3 节（苏格拉底）读完。**

---

## 1. 第一性原理（First Principles）—— Phase 3 用

### 它的本质

不是"从基础讲起"。"基础"是被前人组织过的、已经层层抽象的产物。第一性原理是**比基础还底**——退到这门学问被发明出来之前，问：当年那个人面对的是什么？他不能不假设什么？他一旦假设了某件事，下一步必然推出什么？

终点是：让最终公式 / 定理是被推出来的，不是被宣布的。

### 操作步骤

写 §3 时强制按这个顺序问：

1. **历史动机**（§3.1）：这东西是因为什么具体问题被发明的？如果原书没说，从书里的例题倒推——例题在解决什么实际困难？
2. **最小核心**（§3.2）：把所有装饰、所有一般化、所有变体剥掉，本章其实只在做**一件什么事**？用一句话写出来。
3. **逐层垒砌**（§3.3）：列出最弱的假设。从这个假设能直接推什么？再加一条假设，又能推什么？一直到原书最终公式自然出现为止。

### 检验

写完 §3.3 后，最后一句话应该读起来像 **"……所以我们就拿到了原书 §X.Y 那个公式"**，而不是 **"作者引入了下面这个公式……"**。前者是推导出来的，后者是被宣布的。

### 正例

> **§3.1** 牛顿当年盯着一个尴尬的事实：苹果会落下来，月亮却挂在天上。如果苹果受地球吸引，月亮按理也该掉下来——那它为什么没掉？这不是哲学问题，是会计问题。
>
> **§3.2** 本章其实只在做一件事：**把"苹果掉落"和"月亮绕转"用同一条规则统一起来**。所有 §X.Y 之后的公式，都是这条规则在不同初始条件下的展开。
>
> **§3.3**
> - 假设 1：所有有质量的东西互相吸引（最弱假设——苹果、月亮显然都有质量）。
> - 由此能推出：地球对月亮也有吸引力。
> - 假设 2：吸引力沿连线方向。（这是经济学般的"最简版本"——还能更简单吗？不能。）
> - 由此能推出：月亮被持续拉向地球。
> - 但月亮没掉到地球上 → 一定有什么东西平衡了这个拉力。在直线运动里没有平衡力，所以月亮一定**没在直线运动**……
> - ……（继续推到向心力公式）……
> - 所以我们就拿到了原书 §X.Y 那个 $F = G\frac{m_1 m_2}{r^2}$。

### 反例（不要这样写）

> **§3** 万有引力定律是物理学的基础。它说任意两个物体之间存在吸引力，大小正比于质量乘积、反比于距离平方。这一定律由牛顿在 1687 年提出。

这是教科书摘要，不是第一性原理。它**宣布**结论而不是**推出**结论。用户读完和读原书没区别——这个 skill 就白存在了。

### 当原书顺序和第一性原理冲突时

少数情况下，原书会先讲"什么是 X"，再讲"为什么需要 X"。这种倒装通常是出于**叙述方便**而不是认知必然。在 §3 你按第一性原理重排，**但在 §4 还是按原书顺序逐字讲**。两个阶段的目标不同：§3 要让用户"懂"，§4 要让用户"读得懂这本书"。

## 2. 逆向学习法 —— Phase 2 用

### 它的本质

正常学习是"从知识点出发，希望最终能用"。逆向学习是"从最终能用什么出发，反推哪些知识点是必经之路"。Phase 2 的存在就是为了让用户在还没读 §3-§7 之前就知道**这一切是为了什么**——没有目的地的徒步是耗散心力的。

### 操作步骤

1. 读完整章后，找到所有的**例题、习题、章末复习题、明示的目标**。
2. 把它们翻译成"能力陈述"：**"给定 [输入]，能 [动作]，并且 [质量约束]"**。
3. 列 3–6 条。不要更多——逆向的清单要短才能记住。
4. **必须能对应到原书具体出处**（"对应原书例题 X.Y" 或 "对应习题 Z"）。

### 正例

```markdown
1. 给定一个二元一次方程组（系数任意），能在 5 步内用代入消元法求出唯一解，并能判断方程组是否无解 / 无穷多解。（对应原书例题 3.2、习题 3-1 至 3-4）
2. 看到一组实际问题（如"鸡兔同笼"），能在 1 分钟内列出对应方程组。（对应例题 3.5）
3. 当方程组有 3 个或更多未知数时，能识别"此时代入法效率低、应改用矩阵方法"——并知道矩阵方法是下一章的内容。
```

每条都是**用户读完该章应该当场会做的事**，并且**有书里的题做检验**。

### 反例

```markdown
1. 理解二元一次方程组的概念。
2. 掌握代入消元法的本质。
3. 能够灵活运用所学知识。
```

"理解、掌握、灵活运用"——这些动词不可证伪。用户读完无法判断自己有没有做到。**Phase 2 的能力陈述里禁止使用"理解、掌握、了解、熟悉"这一类不可检验的动词。** 用"能算出"、"能在 N 分钟内"、"能给出反例"、"能识别哪些情形不适用"这种**有动作、有约束**的句式。

## 3. 苏格拉底诘问 —— Phase 6 用

### 它的本质

苏格拉底问答（产婆术）的核心**不是"我问你答"**，是"通过追问让对方自己暴露逻辑漏洞"。在静态课件里没法真追问，所以你能做的是：**预先布下一组问题，每个问题都是一个陷阱或一个使用场景**——用户答得出来证明真懂，答不出来知道自己卡在哪。

### 出题原则（必须遵守）

1. **不出"X 的定义是什么"**——这是鹦鹉学舌，背得出不代表懂。
2. **每题都要求用户使用概念，而不是描述概念。** 比如：
   - 不出："牛顿第二定律是什么？"
   - 改出："一个 2kg 的物体在 5N 的合力下加速度是多少？如果改成 4kg，加速度变化几倍？为什么？"
3. **至少有一道'故意挖坑'题。** 给一个看似合理、其实违反章中假设的命题，让用户判断对错。这是检验"边界感"的唯一方法。
4. **每题答案中要预测错答模式。** 写"如果你答错了，最可能的原因是 X，那是因为你把 A 和 B 混了，回去看 §Y.Z"。这一句比答案本身更有教学价值。
5. **题目难度递进**：Q1 直接调动 §3 的核心 → Q2 调动 §4 的细节 → Q3+ 是边界 / 反例题。

### 题量

4–8 题。少于 4 题不够覆盖，多于 8 题用户会跳过。

### Obsidian callout 语法（一定要用对）

```markdown
> [!question] Q1
> 这里是问题。

> [!success]- 我的回答（先自己想 30 秒再点开）
> 这里是答案。
> 注意 success 后面那个 `-` 是关键，它让 callout 默认折叠。
```

去掉 `-` 就是默认展开，就违背"先想再看"的设计了。**callout 类型可以混用**：question 配 success / hint / warning 都可以，让视觉上有区分。

### 正例

> [!question] Q1
> 我们刚才推出 $F = G\frac{m_1 m_2}{r^2}$ 时，假设了"力沿两体连线方向"。如果我告诉你某个虚构宇宙里引力**不**沿连线方向，而是垂直于连线——你能立刻指出这套理论里至少哪一个结论会失效？

> [!success]- 我的回答（先自己想 30 秒再点开）
> 至少：行星不会做闭合椭圆轨道。因为闭合轨道的存在依赖向心力指向焦点，而焦点正是在两体连线上。如果你回答的是"质量公式会变"——你大概是把"力的方向"和"力的大小"混了。回去看 §3.3 第二条假设。

### 反例

> [!question] Q1
> 万有引力定律的公式是什么？

无意义题。背得出来不代表懂。删掉。

## 4. 费曼讲法 —— Phase 5 用

### 它的本质

费曼说："如果你不能用简单的话讲清楚，你就没真懂。" 反过来用：**Phase 5 是检验你（agent）自己是否搞懂了 §3–§4 的工具。** 写得磕巴 = §3 没穿透 = 回去补 §3。

但费曼讲法对**用户**也有同样的作用——读完 §5 之后用户能不能也讲一遍是终极检验。

### 操作要点

1. **彻底放下书的术语。** 如果原书叫"哈密顿量"，§5 里就叫"系统的总账本"或者"能量管家"——只要类比成立。
2. **用日常物件、生活场景、对话体。** 把抽象概念**落地**到看得见摸得着的东西上。
3. **故意夸张到不严谨。** 严谨是 §4 的任务，§5 的任务是让概念从术语变成直觉。
4. **结尾必须自我打假。** 列出"大白话和原书的差别"——哪些地方为了好懂牺牲了精确？这些差别在做题/考试时会怎么咬人？这一段比类比本身更重要。

### 正例（节选）

> 想象你是个会计，但你管的不是钱，是**能量**。系统里所有粒子的位置和速度构成一本"账本"——这本账本的每一页就是一个"状态"。
>
> 哈密顿量就像账本的**结算规则**：你告诉我现在这一页长什么样，我用结算规则告诉你下一秒这一页会变成什么样。所以薛定谔方程其实在干一件无聊的事——**让账本一页一页翻下去**。
>
> ⚠️ 大白话和原书的差别：
> - 我说"账本"——书里其实是"波函数"，差别在账本是确定的具体数字，波函数是**概率振幅**（一个可以负、可以是复数的"权重"）。这意味着账本可以"自相干涉"，两本账本叠加起来某些页会互相抵消变成 0。**双缝实验考察的就是这个**，你不能用账本类比解释。
> - 我说"结算规则"——书里其实是"算符"。算符的关键性质是它**不一定可交换**（先 A 后 B 不等于先 B 后 A）。账本类比里完全没这个性质。

### 反例

> 哈密顿量是描述系统总能量的算符，它作用在波函数上给出系统的演化。

这是把术语换了换位置，没换皮——**这不是费曼讲法，这是术语翻译**。如果你这一段写不出像样的类比，回 §3 重写——大概率你那里就没真搞懂。

## 5. 四种方法之间的协作

它们不是四个独立模块，是一条认知链：

```
逆向（§2）：知道往哪去
   ↓
第一性原理（§3）：知道每一步为什么是这一步
   ↓
逐字精读（§4）：把每一步落到原书具体文字上
   ↓
费曼（§5）：用大白话压力测试自己的理解
   ↓
苏格拉底（§6）：把测试交给用户做
   ↓
闭环（§7）：用清单核验整条链上有没有断点
```

任何一环弱化或省略，整条链就漏。这就是为什么 SKILL.md 里"不许跳"——每一步都有它在认知链上的位置，跳了就断。
